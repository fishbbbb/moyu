# 总PRD：需求文档（1.0 MVP）

> 说明：本文件为项目总需求文档，覆盖 MVP 范围、用户流程、验收标准与版本规划。各专题 PRD（如阅读条键盘操控模式、[网页提取与书架规范](PRD-网页提取与书架规范.md)）会在本文件中做摘要，并在专题文档中展开。

## 1. 文档概述
### 1.1 背景
用户希望在工作场景下以较隐蔽的方式阅读文本。产品以桌面置顶透明阅读条作为核心形态，并支持本地文件导入与网页内容提取。

### 1.2 目标
- 在桌面提供稳定、可控、可暂停的单行或多行阅读体验
- 支持本地 TXT / EPUB 阅读流程与阅读进度恢复
- 支持网页正文提取并保存到书架
- 采用低成本会员售卖与兑换流程

### 1.3 非目标（1.0 不做）
- 不绕过付费或破解，仅处理用户已合法可见内容
- 不做短信登录、微信登录或第三方登录
- 不做应用商店内购或小程序支付
- 不做云同步（书架与进度均本地）
- 不做目录页批量抓取整本（2.0 再做）

## 2. 用户与核心场景
- 场景 A（本地文件）：导入 txt，自动生成目录，开始桌面阅读，中断后可继续上次进度
- 场景 B（网页阅读）：内置浏览器打开目标网页并完成登录或购买，提取当前页正文，预览确认后保存
- 场景 C（会员）：输入淘宝兑换码开通或续期，客户端显示到期时间

## 3. 产品信息架构
### 3.1 桌面阅读窗口（Overlay）
- 显示：单行或多行，默认 1～6 行；置顶、透明；可配置字体颜色、背景颜色、背景透明度
- 分页：按列数 × 行数将正文重排为固定宽度行，每页容量由设置决定，不按原文换行
- 自动滚动：速度可调，默认暂停，点击开始后滚动，按字/分钟展示
- 交互：点击阅读条显示工具栏；工具栏支持播放/暂停、上一页/下一页、字号调整、设置、关闭
- 进度：支持恢复最近阅读位置
- 快捷键：老板键显示或隐藏

### 3.2 主窗口（配置端）
- 导入与书架：导入 txt、刷新书架、重命名、删除、搜索过滤
- 阅读条设置：外观、速度、翻页步长
- 目录与控制：加载上次位置、开始/暂停、上一页/下一页、关闭
- 采集浏览器：打开网页、提取正文、预览保存、下一章尝试
- 账号与会员：登录/退出、会员到期时间、兑换码

#### 3.2.2 主界面布局重构（工具箱风格）
- 总体结构：`左侧微导航(60-80px) + 右侧上下分栏工作区 + 底部固定工具条`。
- 左侧微导航：仅保留图标入口（书架/网页采集/设置），悬停显示说明，选中态高亮。
- 上半区（核心操作台）：
  - 书架模式：极简搜索栏 + 高密度单列书目；行内操作图标默认隐藏，悬停显示。
  - 网页模式：居中搜索框风格 URL 输入，状态使用胶囊条反馈（提取中/成功/失败）。
- 下半区（详情与控制）：
  - 默认折叠详情，选中书籍或提取成功后展开。
  - 目录使用虚拟列表渲染，保障大目录滚动性能。
- 底部固定工具条：放置“启动阅读条（主按钮）/暂停/设置”等全局动作。
- 视觉策略：去边框化，使用背景微差和轻阴影分层；核心文本深色，辅助文本浅灰。

#### 3.2.1 书架管理（删改查）
- 删除：
  - 在书架列表提供删除入口（悬浮按钮或右键菜单）。
  - 删除书籍时执行级联删除：`book`、`items(章节)`、`progress(进度)` 一并清理，避免脏数据残留。
  - 交互要求：二次确认弹窗（不可恢复）。
- 重命名：
  - 允许用户手动修改书名，修正自动提取产生的站点后缀噪声（如“xxx - 某站名”）。
  - 交互要求：支持在书架列表和详情区两处入口触发重命名。
- 搜索：
  - 在书架顶部提供搜索框，按书名实时过滤（可后续扩展域名/来源过滤）。
  - 在大书库场景下保证筛选响应及时（本地内存过滤，避免频繁 IO）。

## 4. Overlay 行为规范
### 4.1 术语定义
- 行：按当前列数重排后的固定宽度行
- 页：一次可见的 rows 行
- 进度：以重排行数组的 lineIndex 作为存储基准

### 4.2 重排与展示职责
- 主窗口/主进程侧向 Overlay 推送的 `lines` 为“原始行”（按换行切分、保留段落结构）
- Overlay 端在渲染时会按当前 `cols` 对“超长行”做二次切分（避免被裁切），并按 `rows` 展示当前页

### 4.3 配置同步模型
- 以统一配置源为真源，主窗口与 Overlay 共享同一配置
- 修改 rows、cols、fontSize、颜色与速度应即时生效
- 窗口尺寸仅在拖拽缩放或点击应用尺寸时改变

### 4.3.1 Overlay 失焦与“只留文字”模式
- 点击阅读条显示工具栏与缩放手柄时，阅读条四周展示高亮边框与背景（可选透明）。
- 当鼠标点击到 Overlay 之外或切换到其他应用，Overlay 视为失焦：
  - 自动**收起工具栏与设置面板**，保留阅读条文字继续按当前播放状态滚动；
  - 隐藏阅读框边框与背景，仅保留文字本身，尽量降低存在感；
  - 再次点击文字区域会恢复高亮边框与工具栏交互能力。

### 4.4 恢复策略
- 初次加载若无 session，触发恢复最近阅读进度
- 失败时进入空态并显示占位提示

### 4.5 播放与文末行为
- 默认不自动播放
- 到文末自动暂停并停留在末尾

### 4.6 进度写入
- 播放中按时间或行数节流写入
- 暂停、隐藏、退出时强制写入一次

## 6. Overlay 当前实现对齐说明
### 6.1 已实现
- 通过 `overlayGetSession` 获取会话，订阅 `overlayOnSession` 更新
- 按行数组渲染，使用 `whiteSpace: pre`，不让浏览器自动换行
- 空内容时显示占位文案
- 启动时若无 session 尝试 `overlayRestoreLast`
- 支持拖动移动与工具栏显示
- 工具栏功能：播放/暂停、翻页、字号调整、设置、关闭
- 设置面板：外观、行列字号、速度与自动换算
- 自动滚动与进度写入
- 窗口失焦自动收起工具栏与设置面板，并切换为“只留文字”模式（不改变播放状态）

### 6.2 默认值
- rows=1，cols=48
- textColor=#ffffff，bgColor=#000000，bgOpacity=0
- fontSize=16
- autoSpeed=true
- charsPerMinute=2400
- linesPerTick=1

### 6.3 与需求差异
- 默认 CPM：当前实现为 2400（若要对齐可改回 1800）
- linesPerTick：已提供设置入口
- 进度写入：已做节流（播放中按时间/行数节流；暂停/失焦/卸载强制落盘）

## 7. 功能需求
### 7.1 本地文件导入与目录（TXT / EPUB）
需求：导入 TXT / EPUB 后解析为章节目录与行文本流，目录可点击跳转
验收：
- 1MB TXT 可在合理时间内解析，目录跳转后从对应章节开始阅读
- 常规 EPUB（标准容器+OPF+spine）可按章节顺序导入并阅读

#### 7.1.1 当前实现说明（对齐）
- 导入入口在主窗口「导入 TXT / EPUB」。
- TXT：按“章节标题行”启发式识别目录；若命中章节数少于 2，则按「全文」导入。
- EPUB：按 `container.xml -> OPF -> manifest+spine` 抽取章节，章节正文由 HTML 清洗为纯文本。
- 目录点击行为：点击章节标题会把该章加载到阅读条（默认暂停）；继续阅读会按最近进度恢复章节与行号（默认暂停）。
- 存储：书籍与章节（items）入库到 SQLite；进度按 `bookId + itemId + lineIndex` 落盘。

建议补充（后续迭代）：
- TXT 编码兜底（UTF-8/GBK 等自动识别或用户手动选择）
- PDF 适配（先可读、后目录）
- 更丰富的章节规则与“手动目录”入口

### 7.2 网页导入阅读（重构版：阅读模式采集 + 书架化阅读）
需求：用户输入任意相关 URL（简介页/目录页/章节页），系统尽可能识别同一本书并提取可见内容，落库为“书 + 目录 + 章节”，在阅读框中支持上一页/下一页/上一章/下一章/自动阅读。
验收：
- 常见正文页可提取正文并保存
- 输入目录页或章节页时可尽量关联到同一本书
- 目录不完整时允许先导入可见章节并继续补全
- 遇到登录/付费/反爬时有清晰状态与下一步动作

#### 7.2.1 产品范围与前置原则
- 合规原则：仅处理用户当前合法可见内容，不绕过登录、付费、验证码与风控。
- 可用优先：目录不完整可先入库；后续可通过重试/继续提取补全。
- 重复处理：不提供手动“合并书籍”；若识别重复，用户可删除重复书。
- 付费章节策略：自动导航到付费章时默认停驻并提示，不自动跳过。

#### 7.2.2 页面交互设计（主窗口）
1. 导入页（Web Import）
   - 输入框：粘贴 URL（支持简介页/目录页/章节页）
   - 按钮：`打开网页`、`导入本书（智能）`、`仅提取当前页`
   - 状态条：展示当前页面识别状态（可提取/需登录/需购买/疑似风控）

2. 提取进度浮层（Processing）
   - 分阶段显示：
     - 页面就绪中
     - 权限状态检测中
     - 正文提取中
     - 目录识别中
     - 书籍归一中
   - 每个阶段可失败并给出动作建议（登录、购买、刷新重试、切换到目录页）

3. 导入确认页（Preview & Save）
   - 书籍信息：书名（可编辑）、来源站点、来源 URL
   - 当前章节：章节名、正文预览前 N 段
   - 目录信息：已识别章节数、是否完整（完整/部分/未识别）
   - 结果提示：可读章节数、疑似付费章节数（若可判断）
   - 操作：`保存到书架`、`返回网页继续处理`

4. 目录阅读页（Book Detail）
   - 显示章节目录及状态标签：
     - `可读`
     - `需购买`
     - `未抓取`
   - 点击任一章节：从该章进入阅读框

#### 7.2.3 用户流程（全路径）
A. 用户从简介页开始：
1) 打开网页并登录（如需要）
2) 点击“导入本书（智能）”
3) 系统识别书籍信息 + 目录入口 + 当前可读内容
4) 保存后在书架出现该书并附带已识别目录

B. 用户从目录页开始：
1) 打开目录页并导入
2) 系统优先提取目录，再按需提取首章正文
3) 保存后可直接点目录章节阅读

C. 用户从章节页开始：
1) 打开任意章节并导入
2) 系统提取当前章并尝试回溯目录/识别同书章节链
3) 目录缺失时先保存单章，后续继续补全

D. 阅读中遇到付费章：
1) 用户执行“下一章”
2) 命中付费章则停驻提示“本章需购买，前往网页购买后刷新”
3) 用户购买后点击刷新，若可见则从该章继续阅读

#### 7.2.4 提取与归一规则（产品视角）
1. 正文提取优先级
   - L1：结构化数据优先（JSON-LD `articleBody/description/text`、Open Graph/Meta Description）
   - L2：Readability 标准抽取（主引擎）
   - L3：视觉密度兜底（文本密度高、链接密度低的容器评分）
   - L4：手动框选兜底（用户在网页框选后提取选区 HTML 并清洗为纯文本）
   - 说明：系统按 L1→L4 顺序退化，任何一层命中且内容长度达标即停止继续降级。

2. 各层实现细节（技术）
   - L1（结构化数据）：
     - 解析 `script[type="application/ld+json"]`，优先提取 `articleBody`、`description`、`text`、`headline`。
     - 同步读取 `meta[property="og:title"]`、`meta[property="og:description"]`、`meta[name="description"]`。
     - 适用：新闻/博客/CMS 页面元数据完整场景，纯净度高。
   - L2（Readability）：
     - 使用 Mozilla Readability 作为核心正文引擎，处理常规新闻/专栏/博客结构。
     - 与 L1 结果互补：L1 强元数据，L2 强正文结构恢复。
   - L3（Heuristics）：
     - 扫描 `article/main/section/div`，计算候选得分：`文本长度 * (1-链接密度) + 段落数加权 + 句子数加权`。
     - 过滤链接密度过高节点，选取得分最高容器作为正文兜底。
   - L4（Manual）：
     - 自动层失败后，允许用户框选页面区域（或直接文本选中）。
     - 系统抓取选区 DOM 片段，清理脚本/导航/评论/广告节点，再转换为纯文本导入。
  - 性能保护（执行前/执行中）：
    - 脚本执行设置超时（建议 5 秒），超时返回 `EXTRACTION_TIMEOUT`，避免主进程长时间阻塞。
    - DOM 复杂度熔断：当节点数超过阈值（建议 5000）时，返回 `DOM_TOO_LARGE`，提示改用手动框选。
  - 内容清洗策略（执行后）：
    - 样式剥离：提取结果统一移除 HTML 节点的 `style` 与 `class` 属性，避免原站样式污染阅读条。
    - 图片策略（1.0 决策）：默认移除 `<img>`，聚焦纯文本阅读稳定性；如后续提供“显示图片”开关，需提示防盗链可能导致加载失败。

3. 目录识别优先级
   - 页面显式目录（目录页、章节列表区）
   - 上下章链路回溯生成目录
   - 无目录则按“单章书籍”入库并标记可补全

4. 同书识别（避免重复）
   - 站点域名 + 归一化详情页签名 + 书名相似度
   - 命中同书则追加章节；不命中则新建书籍

5. 章节去重
   - 优先 `sourceUrlNormalized`
   - 次级使用标题 + 邻接关系判重

6. 上下章识别补充（URL 相似度推断）
   - 优先使用 `rel="next/prev"` 与锚文本关键词（上一章/下一章/next/prev）。
   - 当缺少明确文案时，启用 URL 路径相似度推断：
     - 同域名、同路径组优先；
     - 识别末尾数字序列增减（如 `/chapter/1 -> /chapter/2`）；
     - 结合路径前缀重合比例计算置信分。
   - 低置信度不自动跳转，仅作为候选提示，避免误跳到评论/推荐页。

#### 7.2.5 阅读框交互约定（与 Overlay 对齐）
- 导入后默认“加载到阅读框但暂停”
- 支持：上一页/下一页/上一章/下一章/自动阅读
- 章末行为：
  - 开启自动下一章：尝试进入下一章
  - 遇到付费/登录受限：停驻并提示
  - 未识别下一章：停驻并提示“手动切换章节”

#### 7.2.6 手动框选（可选兜底能力）
定义：当自动提取失败时，允许用户在网页中手动框选正文区域（或直接选择文本）后保存。
目标：作为 L4 最后兜底，提升复杂页面提取成功率，不保证 100%。
适用：DOM 中有可见文本但自动算法误判；章节页结构混乱；论坛串贴正文被侧栏干扰。
不适用：跨域 iframe、Canvas 文本、强加密动态渲染、未购买内容。

#### 7.2.7 错误码与提示动作
- `AUTH_REQUIRED`：请先在网页中登录，再点击重试
- `PAYWALL_BLOCKED`：本章需购买/订阅，购买后刷新重试
- `ANTI_BOT_OR_BLOCKED`：检测到验证码或访问受限，请在网页完成验证后重试
- `TOC_PAGE_SUSPECT`：疑似目录页（链接列表占比过高），请切换到章节正文页
- `PAGINATION_DETECTED`：检测到分页内容（1/2、2/2 等），请切换单页阅读或翻页后重试
- `FONT_OBFUSCATED`：检测到乱码率过高，疑似字体加密，当前无法稳定解析
- `IFRAME_CROSS_ORIGIN`：正文位于跨域 iframe，请在 iframe 源页打开后再提取
- `SITE_NOT_FOUND`：页面不存在或 404，请检查链接
- `CONTENT_REMOVED`：内容疑似版权下架/删除，请更换来源
- `EXTRACTION_TIMEOUT`：提取执行超时，请切换到正文页或使用手动框选
- `DOM_TOO_LARGE`：页面结构过于复杂，建议使用手动框选
- `NO_MAIN_CONTENT`：未识别到正文，建议切换到章节页或使用手动框选
- `TOC_NOT_FOUND`：未识别到目录，已按单章导入，可后续补全
- `NEXT_CHAPTER_NOT_FOUND`：未找到下一章，请在目录中手动选择

补充交互：网页提取失败默认使用页面内非侵入式错误条和顶部 Toast，不使用阻断式弹窗。
其中 `EXTRACTION_TIMEOUT` / `DOM_TOO_LARGE` 点击错误条可展开“手动框选”兜底动作。

#### 7.2.7.1 边界场景处理流程（实现细节）
1. 登录态检测与引导（`AUTH_REQUIRED`）
   - 检测信号：
     - 页面存在登录表单元素（如 `input[type=password]`、`form[class*=login]`）
     - 页面文本命中“登录/注册/请先登录/sign in/login required”等关键词
   - 处理动作：
     - 停止正文抽取并返回 `AUTH_REQUIRED`
     - UI 提示“请先在网页中登录，再点击提取”
     - 用户完成登录后可直接点击“刷新后重试”

2. 付费墙检测与处理（`PAYWALL_BLOCKED`）
   - 检测信号：
     - 页面文本命中“VIP/付费/购买/订阅/会员/本章需购买”等关键词
     - 命中付费遮罩、模糊层等结构（如 `paywall/vip/subscribe/blur` 类名）
   - 处理动作：
     - 停止正文抽取并返回 `PAYWALL_BLOCKED`
     - UI 提示“本章需购买，前往网页购买后刷新重试”
     - 用户在网页内购买后点击“刷新后重试”

3. 反爬/验证码处理（`ANTI_BOT_OR_BLOCKED`）
   - 检测信号：
     - 页面出现验证码输入框、滑块校验元素、Cloudflare challenge 文本
     - 文本命中“验证码/人机验证/访问受限/verify you are human/challenge”等关键词
   - 处理动作：
     - 停止正文抽取并返回 `ANTI_BOT_OR_BLOCKED`
     - UI 提示“检测到验证码，请在网页中完成验证后重试”
     - 用户完成验证后点击“刷新后重试”

4. 动态渲染/懒加载处理（失败前置增强）
   - 提取前执行页面就绪等待：
     - 等待 `document.readyState=complete`
     - 等待图片加载完成（有超时上限）
   - 懒加载触发：
     - 模拟分段滚动到底部再回顶，触发章节/正文延迟渲染
   - 若仍无法识别正文：
     - 返回 `NO_MAIN_CONTENT`，引导用户切换章节页或使用手动框选（L4）

#### 7.2.8 边界情况清单
- 输入简介页但站点不公开目录：允许导入当前可见内容并提示目录不完整。
- 目录可见但章节正文需付费：目录可展示，未购买章节标记为“需购买”。
- 登录态过期：保留当前书籍与已导入章节，不清空本地数据。
- 网站改版导致规则失效：自动降级至 Readability/DOM 兜底并提示可能不完整。
- 重复导入：优先尝试同书归并；若仍重复，用户可手动删除重复书。
- 登录/购买/验证完成后的统一路径：点击“刷新后重试”，系统重新加载页面并按 L1→L4 再次执行提取。
- 目录误提（内容纯度过低）：当正文中链接密度过高且段落过少时，判定为疑似目录页并中止导入。
- 分页拆分页：检测到“1/2、2/2、下一页”形态时，提示用户切换单页阅读或翻页后重试。
- 字体加密/乱码：当字符异常比例过高（私有区字符/替换字符/异常符号）时，返回“字体加密无法解析”提示。
- 跨域 iframe 正文：当主文档正文过短且主要内容位于跨域 iframe 时，提示改为打开 iframe 源页面。
- 自动阅读防死循环：上下章识别使用 URL 指纹去重，拒绝回到当前页或已访问指纹页面。
- 404 / 版权下架：命中“404/not found/版权下架/已删除”等信号时，返回明确错误码并阻止空内容入库。

#### 7.2.10 网页提取书架与书名规范（专题 PRD 对齐）
> 展开与验收细则见专题 PRD [`PRD-网页提取与书架规范.md`](PRD-网页提取与书架规范.md)；技术调研结论（T1–T4 阶段性结论、工程落点、开放项）见 [`研究-网页导入阅读.md`](研究-网页导入阅读.md) **§18**。

- **书名**：书架列表主行（`books.title`）仅保存**作品本名**（优先 `《…》` 书名号对，去除作者、站点品牌、分类标签等与书名的拼贴）。**来源**以 **`books.domain`** 等在副标题独立展示，不得写入书名。（实现：`desktop/electron/webBookDisplay.ts` 及入库前 IPC 规范化。）
- **目录与简介**：无论从简介页或章节页完成「书籍 + 目录」导入，本地章节表 **`items` 第 1 条**固定为 **「简介」**（写入简介正文；若为空则写入固定占位说明），**第 2 条起**为各章链接行。导入前预览须展示与入库后一致的「简介 + 章节列表」，用于验收 TOC 是否解析成功。（实现：`desktop/electron/db.ts` 中 `importWebBook`；`desktop/src/components/WebImport.tsx`、`MainView.tsx` 预览区。）
- **实现路径（与包级代码的关系）**：当前桌面端用户可见的目录解析主链路为 **`desktop/electron/main.ts`** 中 **`web:extractBookDetail`**（页内脚本 + 可选 adapter）。根目录 **`src/content-extractor/chapterDetector.ts`** 等为算法包资产，**对其单独改动不会自动修复**桌面「解析目录」行为；详见研究文档 §18.4、专题 PRD §5。

### 7.7 阅读条键盘操控模式（K 模式）
需求：在阅读条高频阅读场景下，允许用户以键盘完成上一页/下一页、上一章/下一章、自动阅读开/关等操作，同时让界面在“键盘操控”期间保持干净（默认隐藏工具栏与边框），避免反复点按钮。

范围与原则：
- 非全局快捷键：快捷键仅在阅读条主窗可接收键盘输入且处于 K 模式时生效（避免与其它软件误触冲突）。
- 用显式产品状态替代隐式“焦点猜测”：用户通过工具栏入口开启/退出 K 模式，退出边界清晰可理解。

交互规则（关键）：
- 工具栏新增“快捷键配置”入口与“启用键盘操控”开关。
- K 模式开启后：工具栏窗与阅读条聚焦边框默认收起（可选轻量角标用于提示）。
- 快捷键生效条件：K 模式开启 + 阅读条主窗持有窗口焦点 + 当前无输入控件抢占按键。
- 退出 K 模式：用户在阅读条正文区域单击（非拖拽）退出；拖拽移动阅读条不退出；支持按 `Esc` 退出（建议实现）。
- 从阅读条打开快捷键/设置面板期间：不因 `blur` 自动清除 K 模式（避免用户配置过程中状态被打断）。

验收标准：
- 用户可在工具栏打开快捷键配置，修改键位后无需重启即可生效。
- K 模式开启后，翻页/切章/播放暂停与工具栏按钮行为一致。
- 单击正文退出、拖拽不退出，且打开设置/快捷键面板不会导致 K 模式被静默关闭。

## 9. 风险与约束
- 合规：不绕过付费与权限，仅提取用户可见内容
- 站点兼容：正文抽取失败率存在，需依赖预览确认
- 隐蔽能力：鼠标穿透与任务栏隐藏等平台差异需后置
- 登录态稳定性：部分站点登录态依赖复杂存储/指纹，可能出现“登录后仍不可提取”
- 动态渲染不确定性：SPA/分片渲染页面需等待与滚动触发，导致首次提取耗时波动
- 目录归并风险：同站不同 URL 形态可能指向同一本书，需 URL 归一和站点签名策略降低重复书籍
