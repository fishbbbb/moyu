# 技术设计（1.0 MVP）

## 1. 总体架构
1.0 采用“本地能力为主 + 轻后端”的架构：

- **桌面客户端（Windows/macOS）**：Electron（主窗口 + 桌面阅读 Overlay + 内置浏览器采集）。
- **本地存储**：SQLite（书架、条目、目录、进度、设置）。
- **后端服务**：最小 API（账号、会话、会员、兑换码、更新）。
- **官网**：静态站（介绍/教程/协议/下载/购买引导）。

> 重要：网页账号/购买流程由用户在内置浏览器中自行完成；客户端不收集网站账号密码。

## 2. 客户端模块拆分
### 2.1 主进程（Electron Main）
- 窗口管理
  - `MainWindow`：书架/导入/设置/采集浏览器
  - `OverlayWindow`：桌面阅读窗口（无边框、透明、置顶）
- 全局快捷键
  - 老板键（显示/隐藏 overlay）
  - 说明：当前实现仅注册老板键；其它阅读控制快捷键通过 Overlay 内部（应用内）机制与工具栏/按钮完成
  - 说明：本期新增的「键盘操控模式（K 模式）」快捷键为应用内（非全局）；仅当 OverlayWindow 主窗持有窗口焦点且 K 模式开启时生效
- 与渲染进程通信：IPC（设置、播放控制、读取进度、写入进度）

### 2.1.1 任务栏、程序坞与「低调展示」约束（Windows / macOS）

产品上有部分用户希望**任务栏 / 程序坞尽量不暴露应用**，尤其是 Windows。Electron 与系统能力上可做**有限约束**，无法做到「操作系统层面完全无迹」。

**Windows**

- **`BrowserWindow` 的 `skipTaskbar: true`**：该窗口**通常不再在任务栏显示按钮**。阅读条及附属工具窗当前实现已采用（见 `desktop/electron/main.ts`）。
- **托盘（Tray）**：若主界面隐藏或仅 Overlay 常驻，可配合托盘图标作为入口；任务栏可不再出现应用按钮，但托盘区仍可能有图标（可配置是否显示）。
- **无法由应用单方面保证的**：**Alt + Tab**、**任务管理器**、**Win + Tab** 多任务视图等仍可能暴露窗口或进程——属系统行为，不能视为可完全关闭的「产品开关」。

**macOS**

- **`app.dock.hide()`**：可在 `app.whenReady()` 之后隐藏 **Dock 图标**，适合偏「浮层 / 工具型」形态。
- **`app.setActivationPolicy('accessory')`** 或打包时 **`Info.plist` 中 `LSUIElement`（Application is agent）**：使应用更接近 **Accessory / Agent**，弱化普通前台应用在 Dock 中的存在感；与 **Cmd + Tab、菜单栏** 的交互需按版本实测。
- **菜单栏（状态栏）图标**：可作为唯一入口；若完全不提供托盘/菜单栏入口，用户难以发现应用在运行。
- **无法规避的**：**活动监视器**、**强制退出**等仍列出进程。

**产品与安全上的建议**

- 将「低调」落实为**可选模式**（如设置项），并保留**托盘 / 快捷键 / 老板键**等恢复路径，避免被误判为恶意驻留行为。
- **多窗口**场景下，凡需「不上任务栏」的窗口，需**各自**设置 `skipTaskbar`（或在统一工厂函数中默认开启）。

> 交互约定（1.0 MVP）：导入/选择章节后默认“加载到阅读条但暂停”，由用户显式点击“开始”才自动滚动，避免一打开就乱滚影响使用。

### 2.x 自动阅读速度（单位约定）
- **用户侧单位**：字/分钟（CPM），默认 **2400 字/分钟**
- **内部驱动**：使用定时器间隔 `speedMs`（ms/tick）+ `linesPerTick`（每 tick 前进多少行）
- **换算**：估算每 tick 前进字符数 \(charsPerTick ≈ cols × linesPerTick\)，则
  \[
  speedMs = \frac{60{,}000 \times charsPerTick}{CPM}
  \]
  其中 `cols` 为阅读条设置中的列数（每行字数），用于让速度在不同宽度/行进步长下保持更一致的体感。

### 2.y 进度恢复（退出重进记忆上次位置）
- Overlay 页面初次加载时调用 `overlay:getSession`；若返回空或 `lines` 为空，则调用 **`overlay:restoreLast`**（传入当前配置的 `cols`）。
- 主进程 **`overlay:restoreLast`**：从 DB 取 **最近一条进度**（`progress` 表按 `updatedAt DESC LIMIT 1`），读取该章节 `contentText` 并按**原始换行**切分为 `lines`（`toRawLines`），再根据进度中的 `lineIndex` 恢复 session 并推送到 Overlay；Overlay 端再按当前 `cols` 做“超长行切分”，保证在不同窗口宽度/字号下可稳定显示且不裁切。

### 2.2 渲染进程（UI）
- 书架页：书籍列表、继续阅读、目录面板
- 导入页：本地 TXT / EPUB 导入；网址输入 → 打开采集浏览器
- 采集浏览器页：内置浏览器（WebContents）+ 提取当前页 + 预览确认 + 保存
- 设置页：样式、快捷键配置（含 K 模式开关）、会员入口

### 2.2.1 阅读条键盘操控模式（K 模式）
- 状态：G（普通模式，现网点击阅读条显示工具栏/边框等）与 K（键盘操控模式）
- 入口：工具栏新增“快捷键配置”入口与“启用键盘操控”开关；进入 K 模式后默认收起工具栏窗与聚焦边框/手柄，以保持干净阅读画面
- 快捷键触发条件：
  - K 模式开启
  - OverlayWindow 主窗持有窗口焦点
  - 当前无输入控件抢占按键
- 行为映射（与现有按钮保持一致）：
  - 上一页/下一页：调用 `overlayStepDisplay`
  - 上一章/下一章：调用 `overlayChapterStep`
  - 播放/暂停（自动阅读开关）：调用 `overlaySetPlaying`
- 退出规则：
  - 正文区域单击（非拖拽）退出 K
  - 拖拽移动窗口不退出 K
  - 建议支持 `Esc` 退出
- `blur` 例外：从阅读条打开设置/快捷键面板期间产生的 `blur` 不应导致 K 模式被静默关闭（需与现网 `overlayAuxHideAllSmart` 联动处理）
- 持久化建议：K 模式会话级（关闭 overlay 即退出）；键位映射使用 `localStorage`

### 2.3 内容管线（Content Pipeline）
#### 本地 txt
1. 读取文件（UTF-8/常见编码兜底）
2. 章节识别（多套正则）
3. 行切分（按宽度换行或按固定字数切分）
4. 入库（Book + Items/Chapters + Lines 计算策略）

> 实现对齐：当前 MVP 在渲染进程使用 `FileReader.readAsText` 读取 txt，并用章节标题正则切分为 items；入库在主进程 SQLite（`books/items`）。行切分主要在 Overlay 渲染时按窗口宽度/字号对超长行再切分，避免裁切。

#### 网页导入阅读（D）
1. 内置浏览器打开 URL（用户自行登录/购买）
2. 点击“提取当前页”
3. 从渲染后的 DOM 获取 HTML/文本
4. 正文抽取（Readability 类策略）→ 预览（标题 + 前 N 行 + 域名）
5. 用户确认保存为一个 Item（章节/条目）并生成/关联 Book
6. “下一章”启发式：查找 next 链接/按钮并导航（失败则提示手动）

#### D.1 正文抽取策略（重构版：阅读模式管线）
采用“浏览器阅读模式”思路，将网页导入拆成可观测的多阶段管线：

1. **页面就绪阶段**
   - 等待 `document.readyState === complete`
   - 触发有限次滚动，激活懒加载
   - 记录最终 URL、标题、站点域名、加载耗时

2. **可见性与权限判定阶段（先判定再抽取）**
   - 登录态检测：若正文容器缺失且出现“登录/注册/验证码”等显著文案，返回 `AUTH_REQUIRED`
   - 付费墙检测：若出现“购买本章/订阅/解锁”等遮罩或正文被截断，返回 `PAYWALL_BLOCKED`
   - 反爬/不可读检测：出现验证码、403/429 页面特征，返回 `ANTI_BOT_OR_BLOCKED`

3. **阅读模式抽取阶段（多策略竞赛）**
   - **策略 A：Readability（推荐主策略）**
     - 使用 Mozilla Readability 类算法抽取 `title/byline/content`
     - 优点：通用性强，和浏览器阅读模式理念一致
   - **策略 B：站点适配器（adapter）**
     - 对小说站/内容站做选择器与链接规则适配（如目录识别、章节正文识别）
   - **策略 C：通用 DOM 兜底**
     - `article/main/#content/.content/.entry` 候选容器评分
     - 去噪（导航/评论/推荐/脚本样式）后抽取纯文本
   - 对 A/B/C 结果做评分（长度、段落密度、标题匹配度、噪声比），取最高分结果

4. **结构化阶段（统一书籍模型）**
   - 输出统一结构：`bookMeta + toc + chapter + contentBlocks`
   - 无论来源是 txt 还是网页，都落到同一阅读模型，保证阅读框行为一致

5. **保存与回退阶段**
   - 成功：落库章节正文、目录与来源 URL，更新书架
   - 失败：保留错误码 + 可读提示 + 建议动作（去登录、去购买、手动打开目录页后重试）

> 说明：动态渲染站点支持属于“页面就绪阶段 + 可见性判定阶段 + 抽取阶段”的联合能力，不是单独 UI 功能。

#### D.2 上一章/下一章导航启发式（阅读模式一致化）
优先级：
1. `<link rel="prev|next">`
2. 文本匹配链接（上一章/下一章/上一页/下一页/prev/next/←/→/＜/＞）
3. 站点 adapter 规则（如 `chapterid`、`cid`、`/chapter-123`）
4. URL 递增/递减（仅对明显数字分页，且需同书签名校验）

补充策略：
- 若用户输入任意章节页，先尝试回溯目录页并构建完整 TOC，再定位当前章节在 TOC 中的索引。
- 若无法识别目录，仍允许单章保存并提供“继续尝试识别目录”按钮。

#### D.3 保存策略
- 首次导入：新建 Book，`sourceType=url`，`sourceRef=起始 URL`
- 后续导入：在同 Book 下追加 Item，并递增 `orderIndex`
- 章节标题：优先页面标题，否则用“未命名章节 + 序号”

## 4. 后端 API（最小集）
### 4.1 鉴权
- 使用 `access_token`（JWT 或随机 session token）放在 `Authorization: Bearer ...`
- 1.0 不做多端复杂会话；以“可用”为先

### 4.2 接口列表（建议）
- `POST /auth/register`：邮箱/用户名 + 密码
- `POST /auth/login`
- `POST /auth/logout`（可选）
- `GET /me`：返回会员到期时间等
- `POST /redeem`：输入兑换码开通/续期

- 一码一次：兑换成功后标记已使用并绑定 userId
- 续期：`expireAt = max(expireAt, now) + durationDays`
- 限流：按 IP + userId + code 前缀组合做频控

## 6. 客户端关键实现点（Electron）
### 6.1 置顶透明 Overlay
- `alwaysOnTop: true`
- 透明/无边框：`transparent: true`, `frame: false`
- 建议同时设置透明背景色：`backgroundColor: '#00000000'`（避免 macOS 某些情况下出现不透明底色）
- 跨平台细节：macOS/Windows 对透明窗口渲染不同，需要实际验证；1.0 优先保证可用与性能。

### 6.2 分页与排版（按设置容量，不按原文换行）
- **行切分职责**：
  - 主进程/主窗口推送到 Overlay 的 `lines` 为**原始行**（按换行切分，保留段落结构；见 `toRawLines`）
  - Overlay 渲染时根据当前 `cols` 对“超长行”做二次切分（避免溢出裁切），并按 `rows` 决定一页可见行数
- **每页容量**：由设置中的 **行数（rows）× 列数（cols）** 决定；翻页步长为 `rows` 行，保证“一页”视觉上固定为 rows 行（到文末可能不足）。
- **窄宽度重排修复**：排版列数按阅读框可用像素宽度实时计算，不再强制 `cols >= 20`；因此在窄窗口下也会继续回流，避免最右侧文字被边框裁切。
- **行列边界（当前实现）**：业务侧不再对 `rows/cols` 设固定上限；仅保留基础下限 `>=1`（防止 0/负数导致排版与速度计算异常）。实际可达范围仍受窗口像素尺寸、字体大小、系统窗口管理器与性能影响。
- **窗口**：
  - 移动：使用位于阅读条上缘的一条细窄拖拽带（`WebkitAppRegion: 'drag'`），避免拖拽区域覆盖正文点击与工具栏触发；
  - 缩放：通过边缘/角落把手实现，拖拽缩放时会实时反算 rows/cols，并写回统一配置；窗口最小宽高约束已放开（仅保留 `>=1px` 的技术下限）；
  - 失焦：当 Overlay 失焦时，仅收起工具栏与设置窗并隐藏边框/背景，保留文字继续滚动。

### 6.6 Overlay 辅助窗口（工具栏 / 设置）
- **工具栏与设置**：作为独立窗口（`OverlayToolbarWindow`、`OverlaySettingsWindow`）显示，避免被 Overlay 窗口裁切。
- **定位策略**：工具栏默认跟随 Overlay 右上角；若 Overlay 贴顶导致上方放不下，则工具栏放到 Overlay 下方（避免“压在框内”）。设置窗默认放在 Overlay 下方靠右。
- **状态同步**：主进程对 `overlay:session` 做广播，确保 Overlay/工具栏/设置三窗口的播放状态与进度一致。

### 6.3 网页提取
- 从 WebContents 注入脚本获取 `document.documentElement.outerHTML` 与 `document.title`
- 正文抽取：优先 Readability 类算法（后续可扩展站点规则）
- 预览确认：在保存前展示前 N 行

### 6.4 下一章启发式
优先级建议：
1. `<link rel="next" href="...">`
2. 文本匹配按钮/链接：包含“下一章/下一页/next/→/＞”
3. URL 递增（谨慎，仅对明显的 page=2 或 /2 形式启用）
失败则提示“未找到下一章，请手动打开后再提取”。

### 6.5 配置页与 Overlay UI 约定
- **主窗口配置页**：分组展示——「导入与书架」「阅读条设置（外观 + 速度与翻页）」「书架」「目录与控制（阅读条操作 + 章节）」；阅读条设置与 Overlay 共用同一份配置（localStorage `demo:cfg`），默认白字、无背景。
- **Overlay 工具栏**：点击阅读条后显示，按“播放控制 | 字号 | 设置/关闭”分组，药丸形深色半透明容器，分隔线区分组；关闭按钮悬停高亮为危险色。
- **Overlay 编辑阅读框**：分区为「外观」（文字/背景颜色、背景透明）、「尺寸与排版」（行、列、字号）、「速度」（字/分钟、自动换算、可选 ms/tick）；底部“关闭”“应用尺寸”，主操作为“应用尺寸”。
