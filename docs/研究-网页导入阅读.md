# 网页导入阅读：技术调研与集成指南

## 1) 正文提取算法对比

### Readability（推荐主引擎）
- 库：`@mozilla/readability`
- 维护状态：活跃维护，生态成熟，常用于阅读模式与存档系统。
- 优点：
  - 对新闻/博客/资讯类页面稳定性高
  - 提取结果结构化（标题、内容 HTML、摘要、作者线索）
  - 在 Electron + DOM 环境集成成本低
- 缺点：
  - 对强交互 SPA、短内容页、目录页会误判
  - 对反阅读模式（拆 span、零宽字符）需要二次清洗

### 替代方案
- Mercury Parser（维护较弱，不推荐作为主线）
- postlight/parser（规则依赖较强，通用性略逊）
- 自研启发式抽取（应作为 fallback，不建议替代 Readability）

### 建议
- 主引擎 `@mozilla/readability`
- 失败或正文过短时，启发式 fallback（`article/main/content` 候选评分）

## 2) 动态页面渲染（SPA）

### Electron 原生方案（推荐）
- 使用 `BrowserWindow` / `webContents` 加载目标 URL
- `webContents.executeJavaScript` 在页面上下文执行读取 DOM
- 支持：等待 selector、滚动触发懒加载、超时控制

### 是否需要 Puppeteer
- 1.0 不建议引入：
  - 额外 Chromium 依赖导致打包体积增大
  - 进程与资源管理复杂度更高
- 如果后续需要批处理抓取、无界面任务队列，可评估 Puppeteer（2.0）

## 3) 章节目录识别策略

优先级：
1. `nav/ul/ol` 中带“目录/章节/toc/chapter”等语义
2. `link[rel=next|prev]` 或 `a[rel=next|prev]`
3. 文本匹配“上一章/下一章/previous/next”
4. URL 数字模式推断（如 `/chapter/12` -> `/chapter/13`）

建议：目录识别失败也返回上下章，避免“不可读”。

## 4) 登录态与付费态处理

- 登录态复用：依赖同一 `session` 下的 `cookies`，用户在内置浏览器登录后即可复用。
- 检测策略：
  - 登录关键词：`请登录 / sign in / login required`
  - 付费关键词：`订阅后可读 / members only / subscribe to read`
- 原则：仅提示，不绕过。

## 5) Electron 安全集成建议

- 主进程负责抓取与提取，渲染层仅调 IPC。
- 通过 `preload` 暴露最小必要 API（当前已加 `webExtractStructuredAtUrl`）。
- 长操作（滚动、等待、提取）在主进程执行，避免 UI 卡顿。

## 6) 开源参考结论

- Omnivore：采用 Readability + 站点规则混合策略，适合作为后续扩展路线。
- ArchiveBox：Readability + 存档链路，证明该算法在“可见内容归档”场景可行。
- SimpRead：强调站点适配与阅读模式体验，适合作为规则层策略参考。

## 7) 推荐技术栈

- `@mozilla/readability`：主提取
- `jsdom`：Node/Electron DOM 解析
- `tldts`：域名与 URL 归一
- `fast-levenshtein`：书名相似度去重
- Electron `webContents`：动态渲染 DOM 获取

## 9) 在主窗口中的集成步骤

1. 打开网页：`window.api.webOpen({ url })`
2. 提取正文：`window.api.webExtractStructuredAtUrl({ url })`
3. 预览结果：显示 `content.textContent` / `chapters`
4. 保存：调用现有 `libraryImportWebItem` 或 `libraryImportWebBook`

`MainView` 中 `ensureItemHasContent()` 已使用该结构化接口。

## 10) 示例：从内置网页提取结构化结果

```ts
const res = await window.api.webExtractStructuredAtUrl({ url: 'https://example.com/article' })
console.log(res.content.title)
console.log(res.content.wordCount)
console.log(res.nextChapterUrl)
```

## 11) 测试 URL 与预期

1. BBC 新闻（静态新闻页）
- URL: `https://www.bbc.com/news`
- 预期：正文可提取；目录通常为空；`wordCount > 200`

2. CNN 文章页（静态 + 广告干扰）
- URL: `https://edition.cnn.com`
- 预期：Readability 可过滤导航广告；正文有效

3. Medium（动态渲染）
- URL: `https://medium.com`
- 预期：需等待渲染；滚动后提取成功率提升

4. 知乎专栏（可能登录态）
- URL: `https://zhuanlan.zhihu.com`
- 预期：未登录可能触发 `AUTH_REQUIRED`；已登录可提取

5. 小说目录页（章节链接密集）
- URL: 任一公开小说目录页
- 预期：`chapters.length >= 5`；可识别上下章候选

6. 付费墙页（会员内容）
- URL: 任一订阅文章页
- 预期：触发 `PAYWALL_BLOCKED`，提示购买后重试
