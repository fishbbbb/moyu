# PRD：阅读条键盘操控模式（K 模式）与快捷键配置

> 本文档为专题 PRD。总 PRD 摘要见 `docs/总PRD.md` 的 “7.7 阅读条键盘操控模式（K 模式）”。\n

## 1. 背景与问题

### 1.1 背景

阅读条已支持工具栏按钮：播放/暂停、上一页/下一页、上一章/下一章、字号、设置、关闭。用户在高频阅读时希望**用键盘替代鼠标**，且希望**界面尽量干净**（少占屏幕、少出现边框与工具栏）。

### 1.2 核心矛盾

- **系统「窗口焦点」对用户不可见**：仅靠「焦点在阅读条上」无法向用户解释「为什么有时按键有效、有时无效」。
- **点击与展示耦合**：当前点击阅读条会强化「聚焦态」（如边框）并配合工具栏显示逻辑；若把「第一次翻页键就收工具栏」作为隐式规则，**进入/退出边界不清晰**。

### 1.3 设计立场（本 PRD 采纳）

- **默认策略**：优先 **场景 A**——用户主要在与阅读条交互；快捷键在 **应用内、阅读条持有键盘焦点** 时生效即可，**不将全局系统热键作为默认必选项**（可作为后续阶段）。
- **显式模式**：增加用户可理解的 **「键盘操控模式」**（名称可本地化），用**刻意进入/退出**替代「猜焦点」。

---

## 2. 目标与非目标

### 2.1 目标（Goals）

1. 用户在**少依赖鼠标点击工具栏**的前提下，完成：**上一页/下一页、上一章/下一章、自动阅读开/关**。
2. 提供**可发现的入口**（工具栏内）与**可配置的快捷键**（键位可改、可恢复默认）。
3. 提供**清晰的产品状态**：用户能说明白「当前是否处于键盘操控」以及「如何退出」。
4. 进入键盘操控后，**默认收起工具栏与阅读条边框**（或等价视觉降噪），**不依赖再点一次工具栏**即可持续用键操作。
5. **拖拽移动阅读条位置不退出**键盘操控模式，避免误触退出。

### 2.2 非目标（Non-Goals）

- 本期 **不包含** 默认开启的**操作系统全局热键**（焦点在其它应用时仍翻页）。若未来要做，单独 PRD，并含冲突检测、总开关、默认关闭策略。
- 本期不承诺与**所有**全屏游戏/独占输入场景兼容（全局热键阶段再评估）。
- 不在本期定义「与主窗口书架」的跨窗快捷键（范围仅限 Overlay 相关窗口，见下文）。

## 3. 用户与场景

### 3.1 主要 persona

- **休闲阅读型**：阅读条常置于前台，手眼集中在正文；愿意先点一次进入模式，再长期用键盘。
- **办公副屏型**：阅读条靠边，偶尔切去别的窗口；回到阅读条后通过**显式操作**再次进入或保持习惯即可（本期仍以「焦点在阅读条」为主路径）。

### 3.2 核心场景（User Stories）

1. 作为用户，我希望在工具栏里打开「快捷键配置」，看到**当前键位**并修改，以便符合习惯且不与常用习惯冲突。
2. 作为用户，我希望点击「启用键盘操控」后，**工具栏和边框收起**，仍能用快捷键翻页和开关自动阅读。
3. 作为用户，我希望**再点一次阅读条正文区域**即可**退出键盘操控**，恢复以往「点击出现工具栏/边框」的行为，避免长期卡在「看不见工具栏」的状态。
4. 作为用户，我希望**拖动阅读条**时**不会意外退出**键盘操控。

## 4. 名词与定义

| 术语 | 定义 |
|------|------|
| **阅读条主窗** | 显示正文 Canvas 的透明 Overlay 窗口（`OverlayView`）。 |
| **工具栏窗** | 独立子窗口，展示图标按钮（`OverlayToolbarView`）。 |
| **设置窗** | 阅读框设置面板窗口。 |
| **窗口焦点** | OS 层面当前接收键盘事件的窗口；本功能依赖阅读条主窗（或产品规定的窗口组）获得焦点以派发按键，除非未来引入全局热键。 |
| **键盘操控模式（K 模式）** | 用户显式开启的产品状态：启用本 PRD 定义的快捷键行为；并伴随 UI 策略（默认隐藏工具栏与边框）。 |
| **有效会话** | `overlay` 当前存在 `bookId`/`itemId` 且 `lines` 非空的阅读会话；无会话时快捷键应 noop 或轻提示。 |

## 5. 现状摘要（实现基线）

- 工具栏已通过 IPC 调用：`overlaySetPlaying`、`overlayStepDisplay`（内部与 `readMode`/rows/linesPerTick 一致）、`overlayChapterStep`。
- 阅读条主窗使用 `focused` 等状态控制边框与部分控件显示；`blur` 时会 `overlayAuxHideAllSmart` 收起辅助窗。
- 工具栏按钮集合已通过 `localStorage` key `overlay:toolbar` 配置可见按钮列表。

以上现状约束：**K 模式与失焦收起辅助窗的逻辑必须联合设计**，避免「一点设置就关掉」类回归（见 §8.3）。

## 6. 功能需求

### 6.1 工具栏：入口与配置

1. **快捷键配置**  
   - 在工具栏新增入口（建议图标 + tooltip：「快捷键」或「键盘」）。  
   - 点击后打开**快捷键配置面板**（实现位置二选一，产品可接受任一）：  
     - **A**：在阅读条设置窗内新增「快捷键」Tab（与现有设置统一）；  
     - **B**：独立轻量弹层（仅快捷键），从工具栏打开。  
   - 配置项至少包含以下**可绑定动作**（与工具栏逻辑对齐）：  
     - 上一页、下一页  
     - 上一章、下一章  
     - 播放/暂停（自动阅读开/关）  
   - 每个动作支持：**主键**、可选 **修饰键组合**；支持「恢复默认」；支持「禁用该项快捷键」。  
   - 默认键位（建议初版，可技术评估后调整）：  
     - 上一页 / 下一页：`[` / `]` 或 `PageUp` / `PageDown`（需避免与 IME 强冲突时再调）  
     - 上一章 / 下一章：`Ctrl/Cmd + [` / `]` 或 `Alt + ←/→`  
     - 播放/暂停：`Space`（仅在 K 模式 + 焦点在阅读条时生效，见冲突策略）  

2. **启用键盘操控**  
   - 在「快捷键配置」同区域或工具栏主条提供**显式开关**：「启用键盘操控」。  
   - **开启时**：进入 **K 模式**（见 §7）。  
   - **关闭时**：退出 K 模式（与 §7 退出规则一致）。  

### 6.2 K 模式下的行为

1. **默认视觉**  
   - 收起**工具栏窗**与阅读条**聚焦边框**（或等价：边框保持透明、不显示缩放手柄等「编辑态」装饰——具体以设计稿为准，但原则为**阅读内容优先**）。  
   - 可选：显示**非干扰型指示**（见 §9）。

2. **快捷键生效条件**  
   - **K 模式开启** 且 **阅读条主窗具有窗口焦点** 且 **当前无输入类控件抢占按键**（若未来有搜索框，则焦点在输入框内时，翻页快捷键不触发，方向键留给输入）。  
   - 无有效会话时：不执行阅读动作；可选一次轻提示（toast 或控制台级 debug）。  

3. **与现有按钮逻辑一致**  
   - 翻页、切章、播放暂停的**业务规则**须与 `OverlayToolbarView` 中现有 `stepPage` / `stepChapter` / `overlaySetPlaying` 行为一致（含自动阅读时先暂停再手动翻页等），避免两套逻辑分叉。  

### 6.3 退出 K 模式

1. **主退出路径**：用户在**阅读条主窗正文区域**执行 **单击（pointer up 完成且非拖拽）** → 退出 K 模式。  
2. **拖拽不退出**：若判定为窗口拖动（阅读条移动），**不退出** K 模式。  
3. **辅助退出（建议实现）**：`Esc` 退出 K 模式（与「单击退出」并列）。  
4. **退出后 UI**：恢复现有行为——例如再次单击可重新显示边框/工具栏（与现网逻辑对齐，不在此 PRD 改变失焦收起规则，除非联合评审修改）。  

### 6.4 从工具栏再次进入

- 用户可在工具栏（若仍可通过某种方式唤出，见 §8.3）再次点击「启用键盘操控」，或先退出再进入。  
- **推荐**：K 模式关闭时，工具栏入口与现网一致；K 模式开启时工具栏默认隐藏，**通过「单击退出 K 模式」后**再点阅读条出现工具栏，再进配置——避免在 K 模式下强依赖工具栏。

## 7. 状态机（产品层）

```
[ 普通模式 G ]
  │ 工具栏：启用键盘操控 / 配置中确认启用
  ▼
[ 键盘操控模式 K ]
  │ 条件：session 可有可无；无 session 时快捷键 noop
  │ 阅读条主窗需 focus 才吃键
  │
  ├─ 单击正文（非拖拽） / Esc ──► [ G ]
  ├─ 工具栏「关闭 K 模式」（若可访问）──► [ G ]
  └─ 拖拽移动窗口 ──► 保持 [ K ]
```

**说明**：`G` 与 `K` 是**产品状态**，与 OS focus 独立；但**按键派发**仍依赖 focus（除非未来全局热键）。

## 8. 交互与窗口层级细则

### 8.1 单击 vs 拖拽（退出 K 的判定）

- **拖拽阈值**：沿用或对齐现有窗口拖动判定（例如按下后移动超过 N px 或超过 T ms 视为 drag）。  
- **结论**：拖拽 = 不退出 K；释放后若未构成拖拽 = 可视为点击，**退出 K**（若产品希望「仅单击空白正文退出」而非点拖条也退出，需再拆可点区域，建议 **v1：整个主窗客户区单击均可退出**，拖条区域以拖动为主，若点拖条未移动则退出——需实现时统一手势优先级）。

### 8.2 K 模式下的焦点

- 进入 K 模式时：**阅读条主窗应尽可能获得焦点**（若当前焦点本就在其上则不变）。  
- 用户切换到其它应用：阅读条失焦 → **快捷键不再生效**（符合预期）；**K 模式是否保持**建议策略：  
  - **推荐**：**保持 K 模式**（用户切回阅读条仍可直接按键，无需重开）。  
  - 若失焦即自动退出 K：会导致「切微信回一句回来就要重开」，体验差，**不推荐**作默认。

### 8.3 与 `blur` / 辅助窗收起的关系（关键）

现网：`OverlayView` `blur` 会调用 `overlayAuxHideAllSmart`，收起工具栏与设置窗。

**需求**：从阅读条打开「快捷键配置」若位于**设置窗**，用户点击设置窗时阅读条主窗会 blur —— **不得因此自动退出 K 模式**（若 K 已开启），也不得出现「配置到一半模式被清掉」。

**建议规则**：

- **打开设置/快捷键面板期间**：阅读条主窗 blur **不退出 K**；仅当用户**显式关闭「键盘操控」开关**或**满足 §6.3 退出条件**时退出。  
- 技术实现上可能需要：区分 blur 来源（点到辅助窗 vs 点到其它应用）——与现网 `overlayAuxHideAllSmart` 行为对齐评审。

### 8.4 工具栏窗在 K 模式下的可见性

- **默认隐藏**工具栏窗。  
- **例外**：用户通过系统快捷键或其它入口（若产品保留「始终显示工具栏」设置）时，仍可显示——属可选设置项，**默认关**。

## 9. 视觉与可发现性

### 9.1 轻量指示（推荐默认开启，可在设置里关闭）

- 例如：阅读条一角 **4px 圆点**或**键盘小图标**，半透明；悬停 tooltip：「键盘操控中 · 点击正文退出」。  
- 目的：解决「忘记自己是否还在 K 模式」的问题。

### 9.2 首次进入引导

- 首次开启 K 模式：可选一次性 **非阻塞提示**（2–3 秒消失）：「已开启键盘操控，点击正文或按 Esc 退出」。

---

## 10. 快捷键冲突与输入框策略

1. **仅在阅读条主窗 focus 且 K 模式开启**时，注册/监听自定义快捷键（实现可为 `keydown` + 配置匹配）。  
2. **焦点在可编辑控件内**（未来）：**不触发**阅读快捷键。  
3. **Space**：仅在 K 模式用于播放/暂停时，需评估与滚动/按钮聚焦冲突；若冲突则默认改为 `Space` 仅当未 focus 到其它交互元素时生效，或改用 `P` 作为默认。  
4. **与浏览器/Electron 默认行为**：优先选择 **带修饰键** 的默认组合，减少单键抢占。

| 数据 | 存储位置建议 | 说明 |
|------|----------------|------|

## 12. 无障碍与国际化

- 所有工具栏新按钮提供 `title` / `aria-label`。  
- 文案：简体中文为主；键名显示跟随系统（macOS 符号键）。

## 13. 验收标准（Acceptance Criteria）

1. 用户可在工具栏打开快捷键配置，修改键位后**立即**在阅读条主窗 focus + K 模式下生效（无需重启应用）。  
2. K 模式开启后，工具栏与边框默认隐藏；用户可用配置键完成翻页、切章、播放暂停，行为与工具栏按钮一致。  
3. 用户在阅读条正文**单击**退出 K 模式；**拖拽**移动窗口不退出。  
4. `Esc`（若实现）退出 K 模式。  
5. 从阅读条打开设置/快捷键面板时，**不会因 blur 导致 K 模式被静默清除**（除非用户主动关闭或退出）。  
6. 无有效会话时，快捷键不导致崩溃；表现符合 §6.2（noop 或提示）。
