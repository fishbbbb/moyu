# 用户手册（MVP）

本手册覆盖：**导入 txt → 书架 → 目录 → 桌面阅读条阅读 → 进度恢复**。

**其它文档**：[docs/README.md](./README.md)（含技术设计、开发手册与调研目录索引）。

## 1. 基本概念
- **主窗口**：导入/书架/目录/阅读条控制与设置的入口。
- **桌面阅读条（Overlay）**：置顶透明阅读窗口，显示正文并可自动滚动。
- **章节（Item）**：导入后按“章节标题行”识别切分的条目；若未识别到章节，则只有一个“全文”条目。
- **进度**：以“当前章节 + 行号（lineIndex）”记录最近阅读位置。

## 2. 导入本地 txt
### 2.1 导入步骤
1. 打开主窗口。
2. 在「导入与书架」卡片中点击「导入 txt」，选择一个 `.txt` 文件。
3. 导入成功后，该书会出现在左侧「书架」列表中。

### 2.2 章节目录识别规则（当前实现）
导入时会逐行扫描并尝试命中以下章节标题形式（命中数量 < 2 会视为“未识别到目录”）：
- `第X章/回/节/卷 ...`
- `Chapter N ...` / `CHAPTER N ...`
- `卷 X ...`

若未识别到目录，会提示：**已按“全文”导入**（仍可正常阅读）。

### 2.3 导入失败的常见原因
- **文件内容为空**：会提示“导入失败：文件内容为空。”
- **编码问题**：当前导入使用浏览器侧 `FileReader.readAsText` 读取，若 txt 为非常规编码可能出现乱码（后续可在导入管线增加编码兜底）。

## 3. 开始阅读（Overlay）
### 3.1 打开阅读条（暂停）
推荐流程（避免一打开就自动滚动）：
- 点击 **「继续阅读（暂停）」**：会加载最近进度对应章节到阅读条，并停在上次行号位置。
- 或点击 **「打开阅读条（暂停）」**：只打开阅读条，并按最近进度恢复内容与位置（停在暂停状态）。

### 3.2 从目录加载章节（暂停）
在右侧「目录」中：
- 点击章节标题：**加载到阅读条（暂停）**
- 将鼠标移到某章上：会出现“从本章阅读”等快捷入口（如界面提供）

### 3.3 播放/暂停与翻页
主窗口控制区提供：
- **开始 / 暂停**
- **上一页 / 下一页**（按当前阅读条设置的 `行数 rows` 作为翻页步长）
- **关闭**（关闭阅读条并暂停）

Overlay 自身也支持点击显示工具栏后进行同样操作（播放、翻页、上一章/下一章、字号、设置、关闭）。

> 翻页模式下，自动播放每次按“当前行数 rows”前进；调整阅读框高度或设置中的行数会直接影响每次翻页步长。

## 4. 阅读条设置（外观与速度）
### 4.1 外观
- **字体颜色**、**背景颜色**、**背景透明度**

### 4.1.0 显示与隐私（内容保护）

在阅读条 **设置** 窗的「显示与隐私」中可开启 **内容保护**：由系统**尽力**降低录屏、截图软件拍到阅读条的概率。  
**说明**：不能防止手机拍屏或所有软件；macOS 上部分会议客户端仍可能采集到画面；Linux 上可能无效。详见产品内提示与技术说明 [research/研究-阅读框内容保护.md](./research/研究-阅读框内容保护.md)。

### 4.1.1 尺寸与排版边界
- 阅读框支持拖拽移动与边缘/角落缩放，缩放时会实时重排正文。
- `rows/cols` 不再设置业务上限；输入极端值时，实际显示效果仍受窗口尺寸、字号与系统窗口限制。
- 代码层仅保留 `rows >= 1`、`cols >= 1` 的基础下限，用于避免无效配置。

### 4.2 速度与翻页（字/分钟）
- **字/分钟（CPM）**：越大越快
- **每次前进（行）**：内部每个 tick 前进的行数（影响滚动粒度与体感）

> 速度换算会考虑列数/行数与步长，使不同宽度下更接近一致体感；并有“舒适下限”避免小页翻得过快。

## 5. 快捷键
- **老板键（显示/隐藏阅读条）**：`Ctrl`+`Shift`+`X`（macOS：`⌘`+`Shift`+`X`）

## 6. 进度与恢复
- 播放中会**节流**写入进度；暂停/隐藏/退出等关键时机会**强制写入一次**。
- 重启应用后，阅读条会尝试恢复到**最近阅读**的书籍与章节位置。

## 7. FAQ
- **Q：为什么加载章节后默认不滚动？**  
  A：MVP 约定为“加载到阅读条但暂停”，需要用户明确点击“开始”，避免误触造成内容乱滚。

- **Q：章节识别不准怎么办？**  
  A：当前是正则启发式识别；不准时仍可按“全文”阅读。后续可扩展更多规则或提供手动目录功能。
