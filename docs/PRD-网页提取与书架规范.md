# PRD：网页提取与书架展示规范

> 版本：2026-04-12  
> 状态：**已采纳，实施中**  
> 关联：`desktop/electron/main.ts`（`web:extractBookDetail`）、`desktop/electron/db.ts`（`importWebBook`）、`desktop/src/views/MainView.tsx`、`desktop/src/components/WebImport.tsx`  
> 背景：用户在晋江等站点测试时，书架书名冗长、目录不可见、无法验证提取是否成功；此前对 `src/content-extractor/chapterDetector.ts` 的通用增强**未自动作用于桌面端**（桌面书籍导入走独立页内脚本路径），需在规范层明确「产品真值」后再迭代算法。

> 2026-04-16 实施进展：已完成 `core + adapters` 结构迁移、PASS 一致性门禁、性能分位报告与核心最小自测（`selftest:web-import-core`）。当前严格回归为 18 站点 12 PASS / 6 FAIL，后续按失败站点与 p90 耗时双维度优化。

---

## 1. 问题陈述

1. **书名**：加入书架后展示为「书名 + 作者 + 站点 + 标签」拼贴，违背用户对「书名」的认知；来源信息应独立展示。
2. **目录与简介**：导入后难以确认是否成功解析目录；用户期望**与本地书一致的目录列表**，且**首条为简介**，便于肉眼验收。
3. **界面**：阅读本地书时打开网页导入，预览区仅见简介片段，**看不到章节列表**，与「目录页」心智不一致。

---

## 2. 目标与非目标

### 2.1 目标

| ID | 描述 |
|----|------|
| G1 | 书架上的**书籍名称**仅为作品本名（规范见 §3.1），不得混入作者、站点品牌、分类标签。 |
| G2 | **来源**以独立字段呈现：优先使用已有 `books.domain`（及 UI 副标题），必要时扩展「来源展示名」调研结论见 §6。 |
| G3 | 无论入口为**简介页**还是**章节页**，导入成功后本地目录的**第 1 条**固定为「简介」，内容为该次提取得到的简介正文（或占位说明）；**第 2 条起**为正文章节链接行。 |
| G4 | **导入前预览**与**导入后主界面目录列**一致：可见完整章节列表 + 简介行，用于判断目录是否解析成功。 |

### 2.2 非目标（本期不承诺）

- 不保证任意站点 100% 目录完整（仍允许 `partial` / 失败码）。
- 不为单站写硬编码分支作为**唯一**解法（可与「通用规则 + 低频 adapter」架构并存，见实施手册）。
- 不在本期强制登录/付费墙突破。

## 3. 功能规范

### 3.1 书籍名称（`books.title`）

- **定义**：用户在书架上应看到的「书名」= 作品标题本身。
- **规则（优先级从高到低）**：
  1. 若原始字符串含书名号对 `《…》`，则书名为其中**完整一对**（含书名号）。
  2. 否则去掉尾部站点/标签噪声：如 `【…】` 块、`_站点名_` 中第一段可读标题、`…文学城` 等后缀（实现见 `sanitizeWebBookShelfTitle`，可随调研迭代正则，**不写死单一域名**）。
  3. 若仍无法得到可信短标题，回退为截断后的原文或「未命名网页」，并可在 `warnings` 中记录（后续迭代）。

### 3.2 来源（`books.domain` / 展示）

- **主数据**：继续写入 `domain`（hostname），用于副标题与搜索。
- **展示**：书架列表主行 = 书名；副行 = `domain` 或 `sourceRef`（与现有一致）。**不得**把 hostname 拼进书名。

### 3.3 目录项顺序（`items`）

- `orderIndex === 0`：`title = "简介"`，`sourceUrl` 指向书籍详情/入口 URL（与当前 `detailUrl` 一致），`contentText` 为简介正文；若正文为空，写入**固定占位句**（说明可重试/登录），避免空章导致阅读器异常。
- `orderIndex >= 1`：各章标题 + 章节 URL，正文仍按需抓取（现有逻辑不变）。

### 3.4 导入前预览（UI）

- 在「网页采集 / 书籍模式」成功解析后，除显示书名与章数外，须展示**可滚动短列表**：
  - 第 1 行：简介（可截断预览）；
  - 后续：与将写入书架的章节行一致（至少展示标题，可选 URL tooltip）。
- 阅读本地书时，该预览**不替代**左侧当前书的目录；仅为工具区内嵌预览（与 §4 一致）。

---

## 4. 界面与信息架构

- **书架列表**：一行书名 + 一行来源（domain），与 §3.2 一致。
- **当前书目录列（工具区）**：仅展示**当前选中书**的 `items`；网页导入成功后若选中新书，应看到 **1. 简介 + 2…N 章节**。
- **验收路径**：用户应能通过「预览列表 / 导入后目录」双处确认：简介在首、章节条数合理，无需依赖覆盖层阅读器。

---

## 5. 与「前面更改是否有效」的说明

- **桌面端书籍目录**：依赖 `web:extractBookDetail` 内嵌脚本与（可选）adapter，**不经过**仓库根目录 `src/content-extractor/chapterDetector.ts` 的 Node 链路。
- **因此**：仅改 `chapterDetector.ts` 不会自动修复桌面「解析目录」行为；目录完整度需在 **页内脚本 / 通用 TOC** 上继续迭代（在遵守 §2.2 前提下）。
