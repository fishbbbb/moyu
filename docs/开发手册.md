# 开发手册（Desktop / Electron）

**文档索引**：[docs/README.md](./README.md)。**调研 / 过程日志**：[docs/research/README.md](./research/README.md)。

## 1. 本地启动

```bash
cd desktop
npm install
npm run dev
```

### 1.1 为什么 `npm install` 会比较慢/容易失败
项目依赖 `better-sqlite3`（原生模块），安装后会执行：
- `postinstall`：确保 Electron 版本就绪 + `electron-rebuild` 重编译原生模块

若遇到失败，优先检查：
- 是否安装 Python
- 是否安装系统编译工具链（macOS：Xcode CLI Tools）

可手动重试：

```bash
cd desktop
npm run rebuild:native
```

## 2. 架构速览
### 2.1 进程与窗口
- **MainWindow**：主窗口（书架/导入/目录/控制/设置）
- **OverlayWindow**：桌面阅读条窗口（透明、置顶、可缩放）
- **OverlayToolbarWindow**：工具栏独立窗口（不抢焦点，避免触发 Overlay blur 竞态）
- **OverlaySettingsWindow**：设置独立窗口（可显示/隐藏；自身 blur 时自动收起）

主进程窗口行为与定位策略见：`desktop/electron/main.ts`。

### 2.2 IPC（渲染进程 ↔ 主进程）
渲染进程通过 `window.api` 调用 IPC（由 `desktop/electron/preload.ts` 暴露）：
- Library：`library:listBooks` / `library:getBook` / `library:importTxt` / `library:renameBook` / `library:importWebItem` / `library:importWebBook` / `library:updateItemContent`
- 网页采集（独立 `BrowserWindow`，见 `main.ts` `ensureWebWindow`）：`web:open` / `web:extract` / `web:extractFromSelection` / `web:refresh` / `web:extractStructuredAtUrl` / `web:extractBookDetail` / `web:close`
- Overlay：`overlay:pushSession` / `overlay:resume` / `overlay:getSession` / `overlay:restoreLast` / `overlay:setPlaying` / `overlay:step` / `overlay:chapterStep` / `overlay:hide` / `overlay:setConfig`（`opacity`、`contentProtection` 等）
- Progress：`progress:set`
- Session 广播：主进程通过 `overlay:session` 向 Overlay / 工具栏 / 设置窗同步会话

## 3. 数据存储（SQLite）
数据库文件位置：
- `app.getPath('userData')/app.db`（Electron 用户目录下）

初始化与迁移在 `desktop/electron/db.ts` 中执行（`journal_mode=WAL`，开启外键约束）。

### 3.1 表结构（当前实现）
- `books`：书籍（txt 对应 `sourceType='file'`；网页导入对应 `sourceType='url'`）
- `items`：章节/条目（按 `orderIndex` 排序，正文存在 `contentText`）
- `progress`：阅读进度（以 `bookId` 为主键，存 `itemId` + `lineIndex` + `updatedAt`）
- `settings`：预留（key/value JSON）

## 4. 本地 txt 导入实现
入口在主窗口 UI：`desktop/src/views/MainView.tsx`
- `FileReader.readAsText(file)` 读取内容
- `splitTxtToItems()` 通过章节标题正则将全文切分为 items
- 调用 `window.api.libraryImportTxt({ title, sourceRef, items })` 入库

入库逻辑在：`desktop/electron/db.ts` 的 `importTxtBook()`（事务写 `books` + `items`）。

## 5. 网页导入阅读（需求对照与当前实现）

产品侧完整目标见 **`docs/总PRD.md` 7.2**；以下为与**当前仓库代码**对齐的说明，便于开发与验收时区分「已做 / 部分 / 未做」。  
过程日志、导入集成调研、阅读模式开源对标等见 **`docs/research/README.md`**。

### 5.1 入口与 UI（`desktop/src/views/MainView.tsx`）
- 主窗口提供 **网页模式**，两种子模式（`webMode`）：
  - **`article`（单篇/单章）**：输入 URL → **`打开网页`**（`web:open`）打开独立采集窗 → **`仅提取当前页`**（`web:extract`）或 **`手动框选提取`**（`web:extractFromSelection`）→ 预览后 **`保存到书架`**（`library:importWebItem`，可挂到已有 `bookId`）。
  - **`book`（书籍 + 目录）**：同样先打开网页 → **`导入本书（智能）`**（`web:extractBookDetail`）解析详情页上的目录链接 → 预览章列表 → **`导入书籍目录`**（`library:importWebBook`）。**章节正文默认不入库**，在用户从书架进入章节阅读时再 **按需抓取**（见 5.3）。
- 错误展示：页面内错误文案 + 可展开说明；**`刷新后重试`** 走 `web:refresh` 并按当前 `webMode` 自动再调提取或详情解析。
- 与 PRD 7.2.2 对照：**未实现**独立「分阶段 Processing 浮层」；加载/提取状态主要靠按钮 loading 与 `setNotice`。

### 5.2 主进程与抽取管线（`desktop/electron/main.ts` + `webContentExtractor.ts`）
- **`runStructuredExtraction`**：等待 `WebContents` 加载完成 → `BrowserBridge.triggerLazyLoad` 懒加载预热 → `extractWhenReady` 取 `outerHTML`（带 **超时 5s**、**最大 DOM 节点 5000** 等保护，对应 PRD 性能条款的落地形态）→ **`WebContentExtractor.extractCurrentPage`**。
- **正文策略顺序**（与 PRD 7.2.4 L1→L3 一致思路）：**JSON-LD / meta 结构化文本** → **Readability** → **文本密度启发式容器**；失败则抛错或使用 **手动框选（L4）**。
- **`detectNavigation` / `detectTOC`**：用于结构化接口返回上下章 URL、目录候选等；**Overlay 内翻章**主要仍依赖 DB 与 IPC `overlay:chapterStep`，与站点 next 链路的深度整合以当前代码为准。
- **`web:extractBookDetail`**：在采集窗内执行大段 **`executeJavaScript`**：站点 **adapter**（当前可见 **晋江 `jjwxc`**）+ **泛化目录链接聚合**；若当前页像章节且无目录，会尝试 **`tocUrlCandidate` 自动跳转目录页再解析一次**。

### 5.3 按需拉取章节正文
- 网页书导入后 `items` 可能仅有 `title` + `sourceUrl`，`contentText` 为空。
- `MainView` 中 **`ensureItemHasContent`**：在推送到 Overlay 前若正文为空，则调用 **`web:extractStructuredAtUrl`** 打开 URL 并结构化抽取，再 **`library:updateItemContent`** 写回数据库并刷新本地书目。

### 5.4 错误码（主进程 / 抽取器 → UI）
`webContentExtractor.ts` 中 `ExtractErrorCode` 与 PRD 7.2.7 大量对齐，经 `mapExtractErrorToWebError` 传到渲染进程后由 **`parseWebExtractError`** 解析展示，例如：`AUTH_REQUIRED`、`PAYWALL_BLOCKED`、`ANTI_BOT_OR_BLOCKED`、`TOC_PAGE_SUSPECT`、`PAGINATION_DETECTED`、`FONT_OBFUSCATED`、`IFRAME_CROSS_ORIGIN`、`SITE_NOT_FOUND`、`CONTENT_REMOVED`、`EXTRACTION_TIMEOUT`、`DOM_TOO_LARGE`、`NO_MAIN_CONTENT` 等。  
另有一类 **加载/窗口** 错误在 `main.ts` 中直接构造：`NO_WEB_WINDOW`、`LOAD_TIMEOUT`、`LOAD_FAILED`、`INVALID_URL`、`NO_URL`、`MANUAL_SELECTION_EMPTY`、`MANUAL_SELECTION_TOO_SHORT`、`EMPTY_TOC` 等。

### 5.5 与 PRD 仍存在差距的摘要（验收时注意）
- 无完整「提取进度分阶段浮层」；部分边界码 PRD 已列但产品文案/入口需随迭代核对。
- **目录阅读页（Book Detail）** 独立页：能力多在 **主窗口书架 + 目录面板** 中体现，与 PRD 图示可能不完全一致。
- **章节可读/需购买/未抓取** 等目录状态标签：以当前 DB 与 UI 为准，未必已逐项实现 PRD 7.2.2(4) 的标签体系。

更细的设计叙述见 **`docs/技术设计.md` 第 2.3 节「网页导入阅读」及 D.0 实现状态**。

## 6. Overlay 会话与“恢复上次阅读”
### 6.1 会话模型（OverlaySession）
- `bookId` / `itemId`
- `lines`: string[]（当前章节的原始行；Overlay 端会按实际宽度/字号二次切分超长行）
- `lineIndex`
- `playing`

### 6.2 恢复策略
- Overlay 页面加载时先取 `overlay:getSession`
  - 若为空/无内容，则调用 `overlay:restoreLast`（按 DB 最近进度恢复）
- 主进程 `overlay:resume` / `overlay:chapterStep` 会读取 DB 内容并推送 session
- 进度写入通过 `progress:set` 与若干自动写入点完成（播放/翻页/切章等）

## 6.3 阅读条键盘操控模式（K 模式）实现要点
- K 模式状态主要维护在 Overlay 渲染层（`OverlayView`），并以键盘快捷键监听为触发条件：`K enabled` 且 `session 有效` 且 `主窗持有焦点`。
- 快捷键触发走现有 IPC 调用，避免引入新业务链路：
  - 上一页/下一页：`overlayStepDisplay`
  - 上一章/下一章：`overlayChapterStep`
  - 播放/暂停（自动阅读开关）：`overlaySetPlaying`
- K 模式进入/退出与 UI HUD 联动：
  - 进入 K 后默认隐藏工具栏窗与聚焦边框/手柄（或等价“只留文字”策略）
  - 退出 K：正文单击（非拖拽）或 `Esc`；拖拽移动不退出
- 需要与现网 `blur` 收起逻辑协同：从阅读条打开设置/快捷键面板期间产生的 `blur` 不应导致 K 被自动关闭（需要区分 blur 来源或引入“配置面板打开时的例外标记”）。
- 键位映射默认使用 `localStorage` 存储；K 模式建议会话级（关闭 overlay 即退出），避免用户重进后产生“以为仍在 K”的迷惑。

## 7. 调试建议
- **渲染进程**：开发模式下主窗口会自动打开 DevTools（dock 在右侧）
- **主进程**：可通过终端输出/断点调试 `desktop/electron/main.ts`
- **验证进度恢复**：
  - 导入一本书 → 加载章节到 Overlay → 翻页或播放 → 关闭/重启 → 使用“继续阅读（暂停）”检查位置

## 8. 发行、打包与「解压即用」分发

### 8.1 当前仓库里 `npm run build` 实际产出什么
在 `desktop/` 下执行 `npm run build` 会生成**可被 Electron 加载的资源目录**，**不是**用户双击安装的单一安装包：

- **`dist-electron/`**：主进程与预加载脚本编译结果（如 `main.js`、`preload.js`）
- **`dist/`**：Vite 构建的前端静态资源（如 `index.html` 及静态资源）

> 说明：仓库**尚未**配置 electron-builder / electron-forge 等安装包流水线；若要对外发布安装程序，需在后续迭代中增加打包配置与 CI。

### 8.2 双平台上线时常见的「给用户」的文件形态
在增加 **electron-builder**（或同类工具）后，典型产物为：

| 平台 | 常见产物 | 说明 |
|------|----------|------|
| **Windows** | `.exe` 安装向导（如 NSIS） | 最普遍的「安装版」 |
| **Windows** | **绿色目录打 `.zip`** 或 **portable `.exe`** | 「无需安装、解压即可用」：zip 内为 `win-unpacked` 同类目录，用户解压后运行其中 `.exe`；portable 为单文件便携形态（行为依 builder 配置） |
| **macOS** | `.dmg` | 打开后把 `.app` 拖入「应用程序」等 |
| **macOS** | **`.zip` 内含 `应用名.app`** | 与 Windows 绿色 zip 对应：解压后双击 `.app` 即可运行 |

macOS 分发还需考虑 **代码签名与公证（notarize）**；未签名时用户可能需在「隐私与安全性」中允许或通过「右键 → 打开」首次放行。

### 8.3 原生模块（better-sqlite3）与跨平台出包
项目依赖 **better-sqlite3**，需与 **当前 Electron 版本、目标系统架构** 的 ABI 一致。实务上：

- 在 **Windows 环境**构建 Windows 包，在 **macOS 环境**构建 macOS 包；或  
- 在 CI 中使用 **矩阵构建**（各平台 runner 各打各的包）。

避免在单一平台上交叉产出另一平台的原生 `.node`，否则对端极易启动失败。
