# 研究-网页阅读模式升级路线

## 1. 背景
当前网页导入在“单章正文提取”上已有一定可用性，但在“书籍目录完整性、章节连续阅读、跨站点稳定性”上仍不稳定。  
现状表现为：部分站点仅能抽到第一章、目录不完整、正文尾部噪声较多、错误提示对操作有干扰。

## 2. 目标
- 从“按站点修补”演进为“通用能力 + 重点站点增强”的分层方案。
- 优先保证核心链路：`输入链接 -> 目录 -> 导入 -> 下一章 -> 持续阅读`。
- 在不绕过登录/付费/风控的前提下提升成功率与稳定性。

## 3. 分层能力模型
### L1 通用正文层
- 多策略候选：结构化数据、Readability、密度抽取。
- 统一评分后选最优候选，避免单路径误判。

### L2 清洗与质量层
- 通用去噪：导航、评论、页脚、投票区、二维码引导、版权声明。
- 质量评估：正文长度、段落连续性、链接密度、噪声比。

### L3 导航与续章层
- 综合 `rel=next/prev`、锚文本、URL 递增模式、同域路径相似度。
- 低置信度返回候选而非强制跳转。

### L4 目录层
- 目录抽取独立模块：列表目录、表格目录、脚本数据。
- 章节页自动回溯简介页；目录不完整时记录 `partial_toc` 状态。

### L5 站点增强层
- 对高频站点提供轻量 adapter：
  - 目录源补充
  - 标题清洗
  - 正文尾部噪声裁剪

### L6 可观测与回归
- 输出提取调试摘要（命中策略/候选分/失败码/清洗命中）。
- 建立样本回归集，发布前自动验证。

## 4. 里程碑建议
### 里程碑 A（P0）
- 目录页导入后章节数显著大于 1。
- 从第 1 章可进入第 2 章并成功提取正文。
- 正文尾部无明显操作区噪声。

### 里程碑 B（P1）
- 提取调试摘要可用于快速定位失败原因。
- 通用层对非适配站点可返回“可读正文”而非直接失败。

### 里程碑 C（P2）
- adapter 配置化/插件化。
- 回归样本自动化（章节页/目录页/登录页/付费页/异常页）。
