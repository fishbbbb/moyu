# 研究-网页阅读模式升级路线

## 1. 背景
当前网页导入在“单章正文提取”上已有一定可用性，但在“书籍目录完整性、章节连续阅读、跨站点稳定性”上仍不稳定。  
现状表现为：部分站点仅能抽到第一章、目录不完整、正文尾部噪声较多、错误提示对操作有干扰。

## 2. 目标
- 从“按站点修补”演进为“通用能力 + 重点站点增强”的分层方案。
- 优先保证核心链路：`输入链接 -> 目录 -> 导入 -> 下一章 -> 持续阅读`。
- 在不绕过登录/付费/风控的前提下提升成功率与稳定性。

## 3. 分层能力模型
### L1 通用正文层
- 多策略候选：结构化数据、Readability、密度抽取。
- 统一评分后选最优候选，避免单路径误判。

### L2 清洗与质量层
- 通用去噪：导航、评论、页脚、投票区、二维码引导、版权声明。
- 质量评估：正文长度、段落连续性、链接密度、噪声比。

### L3 导航与续章层
- 综合 `rel=next/prev`、锚文本、URL 递增模式、同域路径相似度。
- 低置信度返回候选而非强制跳转。

### L4 目录层
- 目录抽取独立模块：列表目录、表格目录、脚本数据。
- 章节页自动回溯简介页；目录不完整时记录 `partial_toc` 状态。

### L5 站点增强层
- 对高频站点提供轻量 adapter：
  - 目录源补充
  - 标题清洗
  - 正文尾部噪声裁剪

### L6 可观测与回归
- 输出提取调试摘要（命中策略/候选分/失败码/清洗命中）。
- 建立样本回归集，发布前自动验证。

## 4. 里程碑建议
### 里程碑 A（P0）
- 目录页导入后章节数显著大于 1。
- 从第 1 章可进入第 2 章并成功提取正文。
- 正文尾部无明显操作区噪声。

### 里程碑 B（P1）
- 提取调试摘要可用于快速定位失败原因。
- 通用层对非适配站点可返回“可读正文”而非直接失败。

### 里程碑 C（P2）
- adapter 配置化/插件化。
- 回归样本自动化（章节页/目录页/登录页/付费页/异常页）。

## 6. 完整链路技术方案（输入链接 -> 目录 -> 导入 -> 下一章 -> 持续阅读）

### 6.1 总体架构
- **UI 编排层（Renderer）**：接收用户输入链接、展示目录与章节、触发导入、展示错误与恢复操作。
- **链路编排层（Main Service）**：统一调度 `toc extractor`、`content extractor`、`navigator`、`adapter`，并写入本地库。
- **通用能力层（L1~L4）**：正文抽取、清洗评分、续章识别、目录识别。
- **站点增强层（L5）**：对高频站点注入最小规则，不替代通用链路。
- **可观测层（L6）**：记录每次抽取的策略命中、候选评分、失败码、回退路径。

### 6.2 核心数据结构（建议）
- `BookContext`
  - `inputUrl`: string
  - `normalizedBookKey`: string
  - `site`: string
  - `tocStatus`: `ready|partial|missing`
  - `sourceType`: `book|chapter|unknown`
- `ChapterRef`
  - `title`: string
  - `url`: string
  - `index`: number
  - `confidence`: number
- `ExtractDebug`
  - `contentStrategy`: `structured|readability|heuristic|adapter`
  - `candidateScores`: Record<string, number>
  - `tocSource`: `list|table|script|backtrack|adapter`
  - `cleanRulesHit`: string[]
  - `failureCode`: `none|network|auth_required|paywall|empty_content|blocked|unknown`

### 6.3 端到端时序（主成功路径）
1. 用户输入 URL，UI 调用 `resolveEntry(url)`。
2. 编排层判断 URL 类型（书籍页/章节页/未知）。
3. 若是章节页，先执行 `backtrackToBookPage()` 尝试回溯目录来源页。
4. 执行 `extractToc()`，得到章节列表与 `tocStatus`。
5. UI 展示目录，用户确认导入起始章节（默认第一章或当前章）。
6. 执行 `importChapter(chapterUrl)`：
   - `extractContentCandidates()` 产出多候选正文；
   - `scoreCandidates()` 选最优；
   - `cleanContent()` 做通用去噪 + 站点补充清洗；
   - `saveItem()` 写入本地库。
7. 执行 `resolveNextChapter()` 获取下一章（TOC 优先，导航候选兜底）。
8. 用户点击“下一章”或开启连续阅读，重复步骤 6~7。
9. 每次导入写入 `ExtractDebug`，用于问题定位与回归。

### 6.4 关键算法与回退策略

#### A. URL 分类与归一
- 分类信号：路径模式、查询参数（如 `chapterid`/`novelid`）、页面结构（目录链接密度）。
- 归一规则：去掉追踪参数、锚点与无关分页参数，生成 `normalizedBookKey` / `normalizedChapterKey`。
- 若分类不确定，先按章节页路径执行，失败后自动切到目录模式重试。

#### B. 目录抽取（L4）
- 优先级：`adapter.extractToc` > 结构化目录（脚本/json） > DOM 列表/表格 > 同域链接聚类。
- 质量门槛：
  - 章节数 >= 2 视为可用目录；
  - 若仅 1 章，标记 `partial_toc` 并提示用户切换简介/目录页。
- 去重排序：按 URL 归一键去重；标题相似时保留序号更稳定者；按章序自然排序。

#### C. 正文抽取与评分（L1 + L2）
- 候选来源：`structured`、`readability`、`heuristic`（必要时 `adapter` 提供候选）。
- 评分维度（0~1）：
  - `lengthScore`（有效字符长度）
  - `paragraphScore`（段落连续性）
  - `noisePenalty`（链接密度、按钮词、评论词）
  - `titleMatchScore`（与章节标题相似度）
- 选择策略：取综合分最高候选；若最高分低于阈值，返回 `empty_content` 并附带可解释提示。
- 清洗策略：
  - 通用：去导航/评论/二维码/版权尾巴；
  - 站点：仅附加裁剪，不覆盖通用清洗结果。

#### D. 下一章识别（L3）
- 决策顺序：
  1) TOC 邻接（当前章 index + 1）
  2) `rel=next` / 明确锚文本
  3) URL 递增模式 + 同域路径相似
  4) adapter 补充
- 置信度阈值：
  - `>= 0.75` 自动跳转；
  - `0.45 ~ 0.75` 给候选让用户确认；
  - `< 0.45` 不跳转，提示“返回目录选择下一章”。

#### E. 风控边界（不绕过登录/付费）
- 若命中登录/付费/验证码：
  - 返回状态 `auth_required|paywall|blocked`；
  - UI 给出动作建议（登录后重试、换公开章节、返回目录页）；
  - 不尝试绕过策略，不注入规避脚本。

### 6.5 状态机（建议）
- `idle` -> `resolving` -> `toc_ready|toc_partial|toc_failed`
- `toc_ready|toc_partial` -> `importing` -> `import_ok|import_failed`
- `import_ok` -> `next_resolving` -> `next_ready|next_uncertain|next_missing`
- 任意状态可转 `recoverable_error`（带下一步动作）或 `terminal_error`（需用户更换输入）

### 6.6 接口契约（建议）
- `resolveEntry(url) -> { sourceType, tocStatus, toc[], currentChapterHint, debug }`
- `importChapter({ bookId, chapterUrl, indexHint }) -> { itemId, title, contentText, nextHint, debug }`
- `resolveNext({ bookId, currentItemId }) -> { next: ChapterRef | null, confidence, reason }`
- `retryWithMode({ url, mode: 'toc_first'|'chapter_first' }) -> { ...resolveEntry }`

### 6.7 失败处理与用户体验
- 错误提示必须：
  - 默认自动消失（错误延时更长）；
  - 支持手动关闭；
  - 文案包含“下一步动作”。
- 典型提示模板：
  - `partial_toc`：仅识别到 1 章，建议输入书籍简介/目录页链接重试。
  - `next_uncertain`：下一章不确定，请在目录中手动选择。
  - `paywall`：该章节受付费限制，可切换公开章节继续阅读。

### 6.8 落地排期（两周版本）
- **W1 / P0（先打通）**
  - URL 分类与回溯目录
  - TOC 可用性门槛 + `partial_toc` 提示
  - 正文候选评分与统一清洗
  - 下一章 TOC 邻接优先策略
- **W2 / P1（提稳定性）**
  - next 置信度分级与候选确认
  - 调试摘要落库与导出
  - 高频站点 adapter 最小集（仅补目录与尾部裁剪）

### 6.9 验收用例（最小闭环）
- 书籍目录页输入：章节数 > 1，且可导入第 1 章并定位第 2 章。
- 章节页输入：可回溯到目录或给出 `partial_toc` 可恢复提示。
- 连续阅读：从第 1 章至少可稳定进入第 2 章并抽取正文。
- 噪声控制：正文末尾不出现明显操作区文案（投票/扫码/版权尾巴）。
- 边界场景：登录页/付费页返回明确状态码与下一步动作。

### 6.10 非目标（本阶段不做）
- 不做反爬或登录态绕过能力。
- 不做“全站通杀”站点级深度适配，只做高频站点最小增强。
- 不在本阶段引入复杂分布式抓取与云端渲染。
