# 研究：阅读框内容保护（防录屏 / 截图，尽力而为）

> 实现：`overlay:cfg.contentProtection` → 主进程 `BrowserWindow.setContentProtection`（阅读条主窗 + 工具栏窗 + 设置窗同步）。  
> 产品摘要见 **[总PRD.md](../总PRD.md) §7.8** 与 **[技术设计.md](../技术设计.md) §2.1.2**。

## 机制（Electron）

- API：[BrowserWindow#setContentProtection](https://www.electronjs.org/docs/latest/api/browser-window#winsetcontentprotectionenable)
- **Windows 10 2004+ / Windows 11**：底层多为 `SetWindowDisplayAffinity(WDA_EXCLUDEFROMCAPTURE)`，对 DXGI / Windows Graphics Capture 等常见采集路径通常可隐藏窗口。
- **macOS**：`NSWindow` 共享类型；**不保证**对所有会议软件生效（部分使用 ScreenCaptureKit 或自研采集仍可能拍到）。
- **Linux**：依赖 Chromium / 桌面环境，**可能无效或不稳定**。

## 明确不能防范的情况

- 手机/相机拍屏、外置采集卡
- 高权限恶意软件、部分无障碍/注入类工具
- 部分会议客户端在 macOS 上的采集路径

## 产品表述建议

- 使用「内容保护」「降低录屏/截图可见性」等措辞，避免承诺「绝对防截屏」。
- UI 中已附简短免责说明；本文件供研发与产品复查平台差异时查阅。
