# 主界面工具箱化 UI 设计说明（1.0）

> **状态**：阶段性设计稿。要点已写入 **[总PRD.md](../总PRD.md) §3.2.2** 作为验收摘要；本文件为展开说明。索引：[research/README.md](./README.md)。

## 1. 目标
- 构建“轻量级、高密度、一眼明确”的工具箱体验。
- 默认只展示核心信息，减少边框、分割线和文字噪声。
- 使用非侵入式提示体系，避免打断用户流程。

## 2. 页面结构
- 左侧微导航（60-80px）：书架、网页采集、设置。
- 右侧上半区：当前模式的核心输入和主操作。
- 右侧下半区：目录/预览/详情（按需展开）。
- 底部固定工具条：全局阅读控制动作。

## 3. 模式与交互
- 书架模式：
  - 顶部极简搜索输入（下划线风格）。
  - 单列高密度书目，悬停显示重命名/删除图标。
  - 双击书目直接启动阅读条。
- 网页采集模式：
  - 居中 URL 输入框。
  - 状态胶囊展示提取中/成功/失败。
  - 失败时显示可展开错误条；超时/DOM 过大可直接触发手动框选。
- 设置模式：
  - 展示最常用阅读条参数（颜色、透明度、速度）。

## 4. 提示系统
- 顶部 Toast：
  - 成功提示：2 秒自动消失。
  - 失败提示：常驻，支持动作按钮（如重试）。
- Tooltip：
  - 对复杂按钮用 `title` 或轻提示文案说明（如手动框选）。
- 空状态：
  - 书架空：引导导入入口。
  - 目录空：提示刷新或切换页面重试。

## 5. 性能策略
- 目录区采用虚拟列表渲染（按滚动窗口切片）。
- 搜索输入使用防抖并结合后端查询接口，减少高频 IO。
